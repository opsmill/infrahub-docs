---
title: Override specific Profile values
---

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

When a Profile-assigned object should differ for a specific field, set the value explicitly when creating or updating the object. Explicit values take precedence over Profile values; other attributes still inherit from the Profile.

## Override on creation

<Tabs groupId="method" queryString>
<TabItem value="web" label="Web interface">

1. Navigate to the object list for your node kind
2. Click **Add \<Kind\>**
3. Select the Profile in the **Profile** field
4. Manually fill in the attribute you want to override — the explicit value takes precedence over the Profile
5. Click **Save**

</TabItem>
<TabItem value="graphql" label="GraphQL">

Set the override value alongside the `profiles` reference:

```graphql
mutation {
  <Kind>Create(
    data: {
      name: { value: "<object-name>" }
      <attribute>: { value: <override-value> }
      profiles: [{ hfid: ["<profile-name>"] }]
    }
  ) {
    ok
    object { id }
  }
}
```

</TabItem>
</Tabs>

The object uses `<override-value>` for that attribute and inherits everything else from the Profile.

## Override on an existing object

<Tabs groupId="method" queryString>
<TabItem value="web" label="Web interface">

1. Open the object
2. Click **Edit**
3. Set the attribute value directly in the field
4. Click **Save**

</TabItem>
<TabItem value="graphql" label="GraphQL">

If the object already exists with Profile-inherited values, update the specific attribute:

```graphql
mutation {
  <Kind>Update(
    data: {
      hfid: ["<object-name>"]
      <attribute>: { value: <override-value> }
    }
  ) {
    ok
  }
}
```

</TabItem>
</Tabs>

## Verify

<Tabs groupId="method" queryString>
<TabItem value="web" label="Web interface">

1. Open the object
2. Click the **info** icon next to the overridden attribute

:::success
The metadata shows the value is **not** inherited from the Profile (source is empty).
:::

</TabItem>
<TabItem value="graphql" label="GraphQL">

Query the metadata for the overridden attribute to confirm it is no longer inherited:

```graphql
query {
  <Kind>(name__value: "<object-name>") {
    edges {
      node {
        <attribute> {
          value
          is_from_profile
        }
      }
    }
  }
}
```

`is_from_profile: false` confirms the value was set explicitly and is not inherited from the Profile.

</TabItem>
</Tabs>

## How precedence works

Explicit values on the object beat Profile values. When multiple Profiles also define the attribute, priority decides which Profile would have applied if no explicit value was set — see [Priority and inheritance](./priority-and-inheritance.mdx).
